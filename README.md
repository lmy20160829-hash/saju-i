# 사주아이 saju-i 🐱

복을 부르는 하얀 고양이(마네키네코)가 만세력으로 사주를 봐주는 웹앱입니다.

**사이트: https://lmy20160829-hash.github.io/saju-i/**

- 양력 생년월일·시간·지역을 넣으면 사주 네 기둥(연주·월주·일주·시주)을 한자+한글로 보여줍니다
- 태어난 지역 경도로 **진태양시 보정**을 하고, 년주·월주는 **절기 시각(분 단위)** 기준으로 판정합니다
- 시간을 모르면 시주를 뺀 세 기둥으로 볼 수 있습니다
- 명식표의 글자마다 **십성**(편관·편재·비견 …)을 표시하고, 오행(木火土金水) 분포를 색깔 막대로 보여줍니다
- **대운 타임라인**: 10년 단위 대운 8개(간지·십성·시작 나이)를 가로 타임라인으로 보여주고, 지금 걷는 대운을 강조합니다. **카드를 누르면 그 10년만 깊이 푸는 '선택한 대운' 해설**을 들려줍니다
  - 순행/역행·간지 순서는 lunar-javascript의 Yun, **대운수는 한국 만세력 표준으로 직접 계산**: 절입까지 일수 ÷ 3 반올림(1버림 2올림), 대운수는 세는나이 → 대운 바뀌는 해 = 출생연 + 대운수 − 1. lunar-javascript 자체 유파(起運 날짜의 해)와 1년 어긋나는 사주가 있어 교정했습니다 (기준 사주: 1982-02-24 05:25 여성 → 정유 대운 47세·2028년, `test-core.mjs` 회귀 테스트)
- 계산된 명식만 근거로 한 **AI 해석** — 궁금한 주제를 골라 읽는 **테마별 해설** 방식 (Gemini 무료 API)
  - 풀이마다 맨 위에 **요약 카드**: 키워드 칩 3개 + 기운 지수 게이지 + 행운의 색·숫자·방위·아이템 칩
  - **전체 풀이**: 총평·성품·내면·오행 처방·일·재물·연애·가족·인간관계·당부 10절 (지수는 총운·일직업·재물·연애·인간관계 5항목)
  - **테마 심화 8종**: 재물운 · 연애·결혼운 · 일·직업운 · 건강·생활 리듬 · 인간관계운 · 학업·시험운 · 올해의 운세 · 대운 흐름 — 주제마다 5절(한 줄 요약 → 명리 근거 해설 → 실천 처방)로 한 걸음 더 깊게
  - '올해의 운세'는 서버가 **올해의 세운 간지**(입춘 기준)를 계산해 근거 데이터로 넣어 주고, '대운 흐름'은 계산된 **대운 목록**(간지·나이·시작년)을 검증해 넣어 줍니다 — 지금 걷는 10년을 가장 깊게 풀이
  - 여기에 심화 2종: **선택한 대운**(타임라인 카드 클릭 — 그 10년만, 지난 대운은 돌아보기 관점) · **연도별 운세**(올해 ±10년 중 고른 해 — 그 해가 속한 대운을 배경으로 깔고, 총운 + **재물·연애결혼·일직업·건강리듬·인간관계·학업시험 6분야 상세운**과 분야별 지수 7개로 풀이. 점신 운세보고서의 "총운 → 분야별 상세" 위계 벤치마킹)
  - 한 번 받은 주제는 캐시되어 칩을 다시 눌러도 재요청하지 않고, 생성을 기다리는 동안엔 고양이 로딩 문구가 돌아갑니다
  - **벤치마킹**: 화면 구성(결과 최상단 "내 사주 원국" 명식표 → 아코디언 해설 → 다음 걸음 안내, 로딩 문구 로테이션)은 사주아이(saju-kid)류를, 해설 구성(테마 카테고리, 지수+요약 카드, 행운 아이템)은 점신과 사주아이를 함께 참고했습니다
- **PWA**: 폰 브라우저의 "홈 화면에 추가"로 앱처럼 쓸 수 있습니다

## 실행 방법 (내 컴퓨터에서 보기)

```bash
npm install     # 처음 한 번만
npm run dev     # 개발 서버 실행
```

브라우저에서 http://localhost:8890 접속.
주소 끝에 `?demo`를 붙이면 샘플 사주가 자동 입력됩니다.

```bash
npm test        # 계산 엔진 검증(절기 경계 포함) + 테마별 프롬프트 검증
```

## AI 해석과 API 키 보호

해석은 **Gemini(무료로 시작 가능)** 또는 클로드(Claude) API를 사용합니다.
`.env`에 `GEMINI_API_KEY`(무료 발급: https://aistudio.google.com/apikey)를 넣으면
Gemini를, `ANTHROPIC_API_KEY`를 넣으면 클로드를 씁니다 — 키만 바꾸면 전환됩니다.

어느 쪽이든 **API 키는 절대 브라우저 코드에 넣지 않습니다.**
누구나 소스를 볼 수 있는 정적 사이트에서 키를 넣으면 도용당하기 때문입니다.

```
브라우저 ──(간지 8글자만)──▶ 프록시 서버 ──(키는 여기에만)──▶ Gemini/클로드 API
```

- **내 컴퓨터에서**: `.env.example`을 복사해 `.env`를 만들고 키를 넣으면
  `npm run dev`의 개발 서버가 프록시 역할을 합니다. `.env`는 `.gitignore`에
  등록되어 있어 깃허브에 올라가지 않습니다.
- **배포 사이트에서**: GitHub Pages는 파일만 보여주는 서버라 프록시가 없습니다.
  해석 버튼을 누르면 안내문이 뜨고, 명식표는 정상 동작합니다.
  해석까지 열고 싶으면 `proxy/cloudflare-worker.js`(무료 Cloudflare Worker)를
  배포하고 `js/config.js`의 주소만 바꾸면 됩니다. (파일 안에 순서 설명 있음)
- 브라우저는 프록시에 **간지 한자 8글자와 주제 이름만** 보내고, 프록시는 표에 있는
  글자·주제인지 검증한 뒤 프롬프트를 직접 조립합니다. (프롬프트 조작 방지 —
  주제별 지시문도 전부 서버에만 있습니다)

## 계산 방식 — 왜 라이브러리를 두 개 쓰나요?

사주 간지는 **AI 추정 없이 전부 라이브러리 계산값**을 씁니다.
개발 전 검증에서 라이브러리마다 정확한 부분이 달라, 정확한 쪽을 골라 조합했습니다.

| 담당 | 라이브러리 | 이유 |
|---|---|---|
| 일주·시주·진태양시 보정 | manseryeok-js | 한국 만세력 기준으로 정확 (교차 검증 완료) |
| 년주·월주 (절기 판정) | lunar-javascript | 절기 시각을 천문 계산으로 분 단위까지 정확하게 구함 |

검증 예: 2024년 입춘은 2월 4일 17:27(KST)인데, 17:20 출생은 계묘년·을축월,
17:30 출생은 갑진년·병인월로 정확히 갈립니다. (`test-core.mjs` 참고)

## 폴더 구조

```
index.html            화면 뼈대 (마네키네코 SVG 포함)
css/style.css         디자인 (크림 바탕 + 복빨강/금색)
js/app.js             화면과 계산 엔진을 잇는 코드
js/saju-core.js       사주 계산의 심장 (하이브리드 엔진 + 오행·십성)
js/cities.js          지역별 경도 (진태양시 보정용)
js/interpret.js       테마 칩과 AI 해석 표시 (주제별 캐시 · 로딩 문구)
js/interpret-render.js 풀이 마크다운 → 요약 카드·아코디언 HTML (순수 함수)
js/config.js          해석 프록시 주소 설정
api/interpret.mjs     AI 해석 프록시 (서버 전용, 키는 .env에서 · 테마별 프롬프트)
proxy/                배포용 Cloudflare Worker 프록시 (선택)
manifest.webmanifest  PWA 설정 / sw.js  서비스워커 / icons/  앱 아이콘
vendor/               라이브러리 복사본 + 라이선스
serve.mjs             개발용 미니 서버 (+해석 프록시 내장)
test-core.mjs         계산 엔진 자동 검증
test-interpret.mjs    테마별 프롬프트·검증 로직 자동 검증
test-render.mjs       풀이 렌더러(요약 카드·아코디언·XSS 방지) 자동 검증
verify.mjs            라이브러리 1차 검증 기록 (개발 전 검증용)
```
